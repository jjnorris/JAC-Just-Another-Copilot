# Lite profile

Use this file when you want the smallest JACK subset worth copying into a repository first.

This is an adoption shortcut, not a second canon.
The full repo-scoped layer still lives in `.github/` plus `AGENTS.md`, and the longer canon still lives in `docs/jack/`.

## Minimal repo-scoped files to copy first

Start with:

1. `.github/copilot-instructions.md`
2. `.github/instructions/`
3. `AGENTS.md`

## What that minimal setup gives you

- repo-scoped instructions that travel with the repository
- a small shared operating style for contributors and agent-capable flows
- a base layer you can review and adjust before copying narrower surfaces

## What it does not give you yet

Until you also copy the rest of the pack, you do not get:

- repo-scoped skills under `.github/skills/`
- repo-scoped hooks under `.github/hooks/`
- the repo-scoped custom agent profile under `.github/agents/`
- the longer rules, workflows, templates, examples, and support docs in `docs/jack/`

## When to expand beyond lite

Add the rest of the repo-scoped pack when you want the narrower surfaces to travel with the repository.
Add `docs/jack/` when you want the longer canon, support notes, maintenance inventory, and manual fallback material that support the repo-scoped files.

See `install.md` for transplant steps and `compatibility.md` for support-boundary notes.
